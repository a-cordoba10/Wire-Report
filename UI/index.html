02.
DESIGN REPORT

Color Contrast

#F7F7F7 color which its used in the background of the chat and in some placeholders has a 1.07:1 contrast in comparassion with #FFFFFF, which means it does not have enough contrast for people with visibility impairments.

Video Calls

When performing a video call the application doesn’t ask for permissions, which results in the UI not beeing able to show the call in progress, the user on the other end will recieve the call but won’t be able to see the caller.

03.
PERFORMANCE REPORT

Build

9 .xconfig documents are used to store production and development variables; these variables allow developers to use targets inside the application to dinamically set identity or deployment info values, such as Bundle Identifier or Version

In addition to that, Wire makes use of 2 files: Localizable.strings and Localizables.stringsdict, both are used to dynamically set the language of the app (which Wire has 19) according to the user preferences. Localizable.strings allows to change text within the UI through NSLocalizedString, while Localizable.stringdict defines variables within the application runtime enviroment.

Images

Wire can improve in two aspects about its manage of images. First of all, the project has images outside Images.xcassets folders which is not a good practice, lastly, Wire uses .png where .pdf its the recommended extension, this is because when building the project Xcode will automaticlly resize the images while keeping the properties of vectors in the process

Energy

The process which consumed the most energy was video calling because it used two sensors (camera and micrphone), in relation to this, voice calling was next in energy consumption due to the use of the micrphone, while messaging was last (mainly because it does not use any sensor).

Bugs

The application its smooth and beautiful but it suffers from some bugs that can be easily fix. Next we will address them.

Bad message

The syntaxis of the message its wrong because of the utilization of .StringDict used by string.LocalizedStringWithFormat which doesn't have the numeric value define.

Keyboard

This is due to not executing the method resignFirstResponder() which is part of UITextFieldDelegate. The method notificates an object which has been asked to give up its state as first responder on its window; thus, when the instance of the Video call viewController shows, the keyboard its still present.

Video Call

Video call

When airplane mode its enable after a videocall has begun the application ends the videocall while showing an error message, after recovering connection the application doesn't try to reclaim the call, all of this while the other user is shown a message that the call is paused and still going on.
